面向对象编程

1. 面向对象编程概述

1.1 面向对象编程

OOP（Object Oriented Programming），面向对象程序设计

面向对象与面向过程的区别

面向过程侧重整个问题的解决步骤。着眼局部或者具体实现，如C。

面向对象侧重对象抽象，让对象实现功能，在程序中“万事万物皆对象”。着眼于整体，如C++、C#、Java。

思考：如何将大象装入冰箱？

1.2. 类与对象的关系

举个例子：“学生小张今年22岁，每天学习到10点。学生小李今年18岁，每天学习到12点”。小张和小李都是一个具体的对象，有姓名和年龄的属性，有学习的行为。而对小张和小李的属性和行为的分析，就可以定义出学生这种类型的特点。

类描述了对象的抽象，是创建对象的模板。java中类是引用类型。

对象是类的一个实例，是一个实实在在的个体。

2. package

2.1 包的作用

工程包：用来区分不同项目

语法：公司域名倒写 + 项目名字

com.baidu.search

功能包：用来区分不同代码功能

写法：在项目包下分包，包名不加s

com.baidu.search.db, com.baidu.search.util

2.2 命名规则

包名符合 java 命名规范，并且必须全部小写

不同层级之间用“.”来分隔

2.3 包的声明

声明本类位于哪个包下

使用package声明包，必须放在java文件的首行

package com.bodhixu.java;

2.4 包的导入

声明本类中所有的java类，位于哪个包下

* 表示包下所有的类

如果要使用的类与当前类属于同一个包，则无需导入

java.lang包中的类，无需导入

import java.util.Scanner;

3. 类的定义

3.1 类的组成

类的关键字是class，内部由成员变量和方法组成。

语法：

class 类名{ //类名采用大驼峰式命名

//成员变量(属性)

//方法(行为)

}

3.2 成员变量

成员变量是对属性的描述。变量的数据类型可以是Java的任何数据类型，变量名采用小驼峰式命名。

3.3 方法

方法是对行为的描述。方法名采用小驼峰式命名。

在同一个类中，方法中可以调用自身的成员变量和方法

4. 对象的创建和使用

4.1 对象的创建

声明： 类名 对象名;

实例化(初始化): 对象名 = new 类名();

Student stu = new Student();

4.2 对象调用属性和方法

用“.”运算符来调用

//对象调用属性

stu1.name = “小张”;

stu1.age = 22;

Systemt.out.println(stu1.name);

//对象调用方法

stu1.eat();

stu1.study();

5. 参数

5.1 参数作用

方法运行时，可以通过参数（即方法参数）给方法内部传递数据。

5.2 形参和实参

形参：定义方法时使用的参数。它是形式上的参数，变量名不重要，数据类型重要。

实参：调用方法时实际传给方法的数据。实参的数据类型由形参来决定。

5.3 值传递和引用传递

值传递：传递的是值的副本，即数据值。参数类型包括基本类型和Sring。

引用传递：传递的是引用的副本，即内存地址。参数类型包括非String的引用类型。

5.4 可变参数

可变参数是指参数的数量是可变的。

定义

参数类型后使用…表示可变参数

可变参数必须定义在方法参数列表的尾部，因此一个方法最多只能有一个可变参数

方法内部把可变参数当做数组处理

调用

方法调用时，可以传入0个，1个，多个参数，也可以传入数组或集合。

6. 返回值和返回值类型

6.1 作用

方法运行结束，返回数据给调用者。

6.2 返回类型

返回数据的类型称为返回类型，可以是java中的任意类型。

void是一种特殊返回类型，称为”无返回”。

6.3 返回值

返回的具体数据称为返回值。

通过return返还给调用者的具体数据。

//需求:计算2个数的和

7. 构造方法

7.1 基本语法

方法名和类名相同

没有返回类型

public class Student {

String name;

int age;

public Student(String name, int age) {

​ this.name = name;

​ this.age = age;

}

}

7.2 作用

构造方法的作用在于构造并初始化对象

当实例化对象（new）的时候，系统自动调用

7.3 隐式构造方法

如果类中没有显式定义构造方法，系统会提供一个隐式的无参构造方法

如果类中显式定义了构造方法，则默认的隐式构造方法就不存在了。

如果需要用到无参构造方法，则必须再显式定义。

8. 封装

8.1 面向对象三大特征

封装、继承、多态

8.2 封装性的理解

在进行程序设计时，应尽量避免一个模块直接修改或操作另一个模块的数据，模块设计追求强内聚和弱耦合。封装是把过程和数据包围起来，对数据的访问只能通过已定义的接口（或方法）。

8.3 封装性的实现**

step1：将类的成员变量私有化（使用private声明）

step2：提供setter方法和getter方法供外部访问

9. 继承

9.1 继承的理解

继承可以最大程度实现功能复用，便于项目的维护和管理

9.2 相关概念

子类：实现继承的类称为子类(subclass)，也叫派生类

父类：被继承的类称为父类(superclass)，也叫超类或基类

9.3 继承实现

使用关键字extends

9.4 继承规则

父类中私有的成员变量和方法不能被继承

父类的构造方法不能被继承，但可以调用（后面将super）

Java只支持单继承

Java支持多层继承（java中所有的类，都是继承自Object）

10. 访问权限修饰符

访问权限修饰符可以设置内容的访问权限

修饰符 本类中 本包中 子类中 全局 应用场景
private yes 封装性、本类自己用的
无修饰 yes yes 无特殊需求的
protected yes yes yes 父类被继承的
public yes yes yes yes 工具类等通用的

对象和方法可以使用任意访问权限修饰符

类和接口（后面将）只能有 public 和 无修饰 两种权限

11. 方法重载和方法重写

11.1 方法的重载overload

前提：在同一个类中

规则：一同一不同

一同：方法名相同

一不同：参数列表不同。参数列表不同包括参数类型或参数个数或参数顺序不同。

返回类型、访问权限和异常范围没有限制

普通方法、构造方法都可以重载

11.2 方法的重写（也叫覆写）overwrite （也叫override）

前提：存在继承关系，子类重写父类的方法

规则：二同二小一大

二同： 方法名相同、参数列表相同

二小： 子类返回类型和父类相同或者是父类返回类型的子类，子类中抛出的异常范围比父类小或者与父类相同(后续会讲)

一大： 子类中访问控制权限比父类的大或者相等。

12. this 和 super

12.1 this

调用本类成员变量和方法，说明：一般用于成员变量名称和方法参数名称重名时

this.变量

this.方法(参数)

调用本类构造方法，必须放在的该构造方法的首行

this(参数)

表示当前对象的内存地址。

12.2 super

调用父类的成员变量和方法，用户和this一致

调用父类构造方法，用户和this一致

13. final

final翻译成中文是“最终的”意思，可以修饰变量、方法和类，优势在于可以提高访问速度。

13.1 修饰变量

final修饰的变量，称为常量。常量只能被赋一次。

final一般和static配合使用定义状态码，如public static final int STATE_LOADING = 10;

13.2 修饰方法

final修饰的方法，不能被重写

13.3 修饰类

final修饰的类，不能被继承

14. static

static 翻译成中文叫“静态的”，可以修饰变量、方法、构造代码块。静态事物独立于类的实例（即对象），所有对象中的静态事物共享一块存储区。

14.1 静态变量

定义：用static修饰的成员变量，叫静态变量，也叫类变量。如：public static int a;

普通的成员变量，叫做实例变量。

访问：通过类名访问静态变量

应用场景：一般和final一起使用，表示静态常量

public static final DAYS_IN_WEEK = 7; //1周7天

14.2 静态方法

定义：用static修饰的方法，叫静态方法，也叫类方法。

静态方法中只能调用静态变量和静态方法

静态方法中不能使用this和super

static方法不能被重写

应用场景：一般工具类中的方法使用static

14.3 静态构造代码块

概念

构造代码块：在类中直接用一对大括号括起来的一段代码

静态代码块：用static修饰的构造代码块称之为静态代码块

区别

构造代码块在每次new对象的时候都会执行一次

静态代码块只有在类第一次加载到内存的时候执行一次

应用场景：一般执行一次的初始化操作放在静态构造代码块中

14.4 注意事项

静态事物一旦分配内存，则无法手动回收，直到程序结束。因此非必要时尽量避免使用。

15. 抽象

15.1 抽象的理解

只规定了行为，但不规定具体的实现。比如父类Person中只规定了吃的行为，至于怎么吃由子类Student和Teacher具体决定。

15.2 抽象方法

使用abstract修饰，没有方法体（没有方法实现）

public abstract void eat();

15.3 抽象类

使用abstract来修饰

类中如果出现了抽象方法，则该类必须标记为抽象类

抽象类中可以没有抽象方法

抽象类中可以有普通类中有的任何元素 (成员变量、静态变量、实例方法、静态方法、构造方法等)

不能实例化抽象类的对象，抽象类必须被继承使用

子类继承抽象类时，或者实现父类的抽象方法，或者子类也标记为抽象类。

16. 接口

16.1 接口的理解

抽象类不但允许有抽象方法，还可以有成员变量和普通方法等。如果抽象的更彻底，只留下常量和抽象方法，则需要使用接口。

接口可以理解为对行为或规范的定义。面向对象的最高境界，是面向接口编程！！！（更好的扩展性和较低的耦合性）

16.2 接口的声明

[权限修饰符] interface 接口名

关键字 interface

访问权限修饰符和类一样，只有public和无修饰两种

接口名一般以I或On开头、以able或者Listener结尾

16.3 接口内部组成

JDK8之前，接口内部只能包括公有的静态常量和公有的抽象方法，JDK8对接口进行了扩展（后面讲解）。

修饰符可以完全省略，自动编译时自动添加

16.4 接口的父接口

接口可以继承多个父接口，用extends。

16.5 接口的实现

关键字 implements

一个类可以实现多个接口，多个接口之间使用逗号隔开。

实现类必须实现所有接口的所有抽象方法，如果不能完全实现，则该类必须标记为抽象类。

一个类实现多个接口的同时，还可以来继承一个类。必须先写继承，再写实现。

17. 多态*

17.1 编译时类型和运行时类型

编译时类型：编译时类型由声明该变量时使用的类型决定(赋值运算符左边)

运行时类型：运行时类型由实际赋给该变量的对象决定(赋值运算符右边)

Persont person = new Student();

17.2 多态的概念

父类引用指向子类对象，又称为编译时类型和运行时类型不一致

17.3 自动类型转换和强制类型转换

自动类型转换：父类引用指向子类对象

Persont person = new Student();

父类引用只能直接调用父类中存在的方法，不能调用子类中新增的方法(否则会出现编译错误：找不到方法)，因为此时的编译时类型是父类类型

强制类型转换：父类引用强转成子类对象

Student stu = (Student)p;

强制类型转换有风险，如果类型不匹配就会产生异常ClassCastException

17.4 instanceof

类型判断，判断对象是否属于该类型

语法：对象名 instanceof 数据类型

18. 内部类

18.1 概念

将一个类定义在另一个类的里面，对于里面那个类就称为内部类（内置类，嵌套类）。内部类包括成员内部类、匿名内部类、局部内部类、静态内部类等。

18.2 成员内部类

优点：可以自由访问外部类的成员

缺点：破坏了类的结构，降低了代码可读性

19. 枚举

19.1 概念

enum（ enumeration），枚举的定义是为了将一组相关常量统一管理。实际开发中的错误码、状态码等一般使用枚举定义。

19.2 声明和调用

19.3 常用方法

Java 中的每一个枚举都继承自 java.lang.Enum 类。当定义一个枚举类型时，每一个枚举类型成员都可以看作是 Enum 类的实例，这些枚举成员默认都被 final、public, static 修饰。如果枚举不添加任何方法，枚举类型成员的枚举值默认为从0开始的有序数值。

values()：返回 enum 实例的数组，而且该数组中的元素严格保持在 enum 中声明时的顺序。

name()：返回实例名。

ordinal()：返回实例声明时的次序，从0开始。

equals()：判断是否为同一个对象。也可以使用==判断，效果是一致的。

System.out.println(Color.values()[1]);

Color color = Color.GREEN;

System.out.println(color.name());

System.out.println(color.ordinal());

Color color2 = Color.GREEN;

System.out.println(color.equals(color2));

System.out.println(color == color2);

声明枚举时，所有的值必须写注释

枚举可以用于方法参数类型，但是不建议作为方法返回类型
