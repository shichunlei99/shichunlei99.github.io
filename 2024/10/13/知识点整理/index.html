知识点整理

标识符命名规则

由字母、数字、下划线（_）和美元符号（$）组成
不能以数字开头
区分大小写
不能是Java中的关键字和保留字

float，后面必须加F或者f

double，后面可以加D或d或者不加（即浮点类型默认是double类型）

浮点型精度不准确

字符类型（char）
char和int互相转换

自动类型转换

强制类型转换

算术运算符

+、 - 、 * 、 / 、 % 、 ++ 、 –

赋值运算符
**= 、 += 、 -= 、 = 、 /= 、 %=*

关系运算符

关系运算符又称为比较运算符，是比较两个数的大小，运算结果为布尔类型。
>、 < 、 >= 、 <= 、 == 、 !=

逻辑运算符

逻辑运算符是比较两个布尔类型数据的逻辑关系，运算结果依旧是布尔类型。
&、| 、! 、^、&&、 ||

练习

打印九九乘法表

  1. 外层循环：for (int i = 1; i <= 9; i++) 控制行数，从1到9。
  2. 内层循环：for (int j = 1; j <= i; j++) 控制列数，从1到当前行数i。
  3. 打印乘法表项：System.out.print(j + "*" + i + "=" + (i * j) + "\t"); 打印当前的乘法表项，并使用制表符\t来对齐。
  4. 换行：System.out.println(); 在内层循环结束后换行，开始下一行的打印。

数组排序冒泡排序

每位数字依次和后面的数字比较，小的互换位置。

优点：简单

形参和实参

形参：定义方法时使用的参数。它是形式上的参数，变量名不重要，数据类型重要。

实参：调用方法时实际传给方法的数据。实参的数据类型由形参来决定。

值传递和引用传递

值传递：传递的是值的副本，即数据值。参数类型包括基本类型和 Sring。

引用传递：传递的是引用的副本，即内存地址。参数类型包括非 String 的引用类型。

void 方法碰到 return 会自动结束方法

构造方法作用

构造方法的作用在于构造并初始化对象

当实例化对象（new）的时候，系统自动调用

封装性的实现

step1：将类的成员变量私有化（使用 private 声明）

step2：提供 setter 方法和 getter 方法供外部访问

继承的理解

继承可以最大程度实现功能复用，便于项目的维护和管理

子类：实现继承的类称为子类 (subclass)，也叫派生类

父类：被继承的类称为父类 (superclass)，也叫超类或基类

访问权限修饰符

访问权限修饰符可以设置内容的访问权限

修饰符 本类中 本包中 子类中 全局 应用场景
private yes 封装性、本类自己用的
无修饰 yes yes 无特殊需求的
protected yes yes yes 父类被继承的
public yes yes yes yes 工具类等通用的

方法的重载 overload

前提：在同一个类中

规则：一同一不同

一同：方法名相同

一不同：参数列表不同。参数列表不同包括参数类型或参数个数或参数顺序不同。

返回类型、访问权限和异常范围没有限制

普通方法、构造方法都可以重载

方法的重写（也叫覆写）overwrite （也叫 override）

前提：存在继承关系，子类重写父类的方法

规则：二同二小一大

二同： 方法名相同、参数列表相同

二小： 子类返回类型和父类相同或者是父类返回类型的子类，子类中抛出的异常范围比父类小或者与父类相同 (后续会讲)

一大： 子类中访问控制权限比父类的大或者相等。

this

调用本类成员变量和方法，说明：一般用于成员变量名称和方法参数名称重名时

super

调用父类的成员变量和方法，用户和 this 一致

调用父类构造方法，用户和 this 一致

final

final 翻译成中文是 “最终的” 意思，可以修饰变量、方法和类，优势在于可以提高访问速度

static

static 翻译成中文叫 “静态的”，可以修饰变量、方法、构造代码块。静态事物独立于类的实例（即对象），所有对象中的静态事物共享一块存储区。

静态变量

定义：用 static 修饰的成员变量，叫静态变量，也叫类变量。如：public static int a;

普通的成员变量，叫做实例变量。

访问：通过类名访问静态变量

应用场景：一般和 final 一起使用，表示静态常量

静态构造代码块

概念

构造代码块：在类中直接用一对大括号括起来的一段代码

静态代码块：用 static 修饰的构造代码块称之为静态代码块

区别

构造代码块在每次 new 对象的时候都会执行一次

静态代码块只有在类第一次加载到内存的时候执行一次

应用场景：一般执行一次的初始化操作放在静态构造代码块中

静态方法

定义：用 static 修饰的方法，叫静态方法，也叫类方法。

静态方法中只能调用静态变量和静态方法

静态方法中不能使用 this 和 super

static 方法不能被重写

应用场景：一般工具类中的方法使用 static

静态事物一旦分配内存，则无法手动回收，直到程序结束。因此非必要时尽量避免使用。

抽象的理解

只规定了行为，但不规定具体的实现

抽象方法

使用 abstract 修饰，没有方法体（没有方法实现）

抽象类

使用 abstract 来修饰

接口的理解

抽象类不但允许有抽象方法，还可以有成员变量和普通方法等。如果抽象的更彻底，只留下常量和抽象方法，则需要使用接口。

接口可以理解为对行为或规范的定义。面向对象的最高境界，是面向接口编程！！！（更好的扩展性和较低的耦合性）

接口的声明

[权限修饰符] interface 接口名

关键字 interface

访问权限修饰符和类一样，只有 public 和无修饰两种

接口名一般以 I 或 On 开头、以 able 或者 Listener 结尾

接口的实现

关键字 implements

多态

编译时类型和运行时类型

编译时类型：编译时类型由声明该变量时使用的类型决定 (赋值运算符左边)

运行时类型：运行时类型由实际赋给该变量的对象决定 (赋值运算符右边)

instanceof

类型判断，判断对象是否属于该类型

语法：对象名 instanceof 数据类型

内部类

将一个类定义在另一个类的里面，对于里面那个类就称为内部类（内置类，嵌套类）。内部类包括成员内部类、匿名内部类、局部内部类、静态内部类等。

枚举

enum（ enumeration），枚举的定义是为了将一组相关常量统一管理。实际开发中的错误码、状态码等一般使用枚举定义。

toString()

该方法的返回值描述了当前对象的有关信息

Character

boolean isDigit(char ch) 判断字符 ch 是否为 0~9 之间的数字
boolean isUpperCase(char ch) 判断字符 ch 是否为大写形式
boolean isLowerCase(char ch) 判断字符 ch 是否为小写形式

BigDecimal 和 BigInteger

BigDecimal 类位于在 java.math 包中提供的，用来对超过 16 位有效位的浮点数进行精确的运算。BigInteger 与 BigDecimal 类似，可以操作长度超过了 long 的数据。这些大数可以以字符串的形式传入。

public String toUpperCase () 把字符串中的英文字符全部转换为大写字符，返回值为转换后的新的字符串。
public String trim () 把字符串中的首尾的空白字符去掉
public String replace (CharSequence target, CharSequence replacement) 使用指定的字面值替换序列替换此字符串所有匹配字面值目标序列的子字符串
public String replace (char oldChar, char newChar) 返回一个新的字符串，它是通过用 newChar 替换此字符串中出现的所有 oldChar 得到的。
public String replaceAll (String regex, String replacement) 使用给定的 replacement 替换此字符串所有匹配给定的正则表达式的子字符串。
public replaceFirst (String regex, String replacement) 使用给定的 replacement 替换此字符串匹配给定的正则表达式的第一个子字符串。
public String [] split (String sign) 返回一个数组，该数组由目标字符串根据 sign 拆分得到
public char [] toCharArray () 将字符串拆分成字符数组

数组和集合的区别

数组：长度是不可改变的，可以存放任意类型的数据。
集合：大小是可变的，只能存引用类型的数据。

两大家族

Collection (接口), 存储的是单一对象。
Map (接口), 存储的是 key-value (键值对) 形式的数据。

三个辅助

Iterator (迭代器), 主要用于遍历 Colleciton
Collecitons (工具类),Colleciton 集合的工具类
Comparator/Comparable (比较器), 用于对象之间的比较排序

异常

异常是程序在编译或运行期间发生的不正常的事件。异常会打断指令的正常执行流程，如 int i = 1/0。

常见的编译时异常

java.lang.ClassNotFoundException：没有找到指定名称的类

java.io.FileNotFoundException ：访问不存在的文件

java.io.IOException：操作文件时发生的异常

java.sql.1：操作数据库时发生的异常

ava.lang.ClassCastException：错误的类型转换

java.lang.ArrayIndexOutOfBoundsException：数组下标越界

java.lang.NullPointerException：空指针访问

java.lang.ArithmeticException：算术异常 (除 0 溢出)

常见的运行时异常

java.lang.ClassCastException：错误的类型转换

java.lang.ArrayIndexOutOfBoundsException：数组下标越界

java.lang.NullPointerException：空指针访问

java.lang.ArithmeticException：算术异常 (除 0 溢出)

final、finally、finalize的区别。

final用于声明变量、方法和类的，分别表示变量值不可变，方法不可覆盖，类不可以继承
finally是异常处理中的一个关键字，表示finally{}里面的代码一定要执行
finalize是Object类的一个方法，在垃圾回收的时候会调用被回收对象的此方法。

3、throw 抛出Exception
4、throws 方法里面表示要抛出那些exception
