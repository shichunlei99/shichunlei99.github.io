JDBC 简介

JDBC（Java Database Connectivity）是 Java 语言中用于连接数据库和执行 SQL 语句的一套 API。它允许 Java 程序与各种关系型数据库进行交互，实现数据的存储、检索、更新和删除等操作。

二、JDBC 工作原理

  1. 加载数据库驱动

    • 在使用 JDBC 之前，需要先加载特定数据库的驱动程序。这通常通过使用 Class.forName() 方法来实现，传入数据库驱动类的全限定名。例如，对于 MySQL 数据库，可以使用 Class.forName("com.mysql.cj.jdbc.Driver")。
    • 加载驱动后，DriverManager 会自动注册该驱动，以便后续建立数据库连接。

  2. 建立数据库连接

    • 使用 DriverManager.getConnection() 方法建立与数据库的连接。该方法接受三个参数：数据库连接 URL、用户名和密码。
    • 连接 URL 的格式取决于具体的数据库。例如，对于 MySQL 数据库，连接 URL 可能是 jdbc:mysql://localhost:3306/mydatabase，其中 localhost 是数据库服务器的地址，3306 是端口号，mydatabase 是要连接的数据库名称。
    • 建立连接后，会返回一个 Connection 对象，代表与数据库的连接。

  3. 创建 Statement 或 PreparedStatement 对象

    • Connection 对象提供了创建 Statement 和 PreparedStatement 对象的方法。
    • Statement 用于执行静态 SQL 语句，而 PreparedStatement 用于执行预编译的 SQL 语句，它可以防止 SQL 注入攻击，并且在执行多次相同的 SQL 语句时性能更好。

  4. 执行 SQL 语句

    • Statement 和 PreparedStatement 对象都提供了执行 SQL 语句的方法。
    • 对于查询操作，可以使用 executeQuery() 方法，它返回一个 ResultSet 对象，代表查询结果集。
    • 对于插入、更新和删除操作，可以使用 executeUpdate() 方法，它返回一个整数，表示受影响的行数。

  5. 处理结果集（如果是查询操作）

    • 如果执行的是查询操作，需要遍历 ResultSet 对象来获取查询结果。
    • ResultSet 提供了一系列方法来获取不同类型的数据，如 getInt()、getString() 等。可以使用 next() 方法来逐行遍历结果集。

  6. 关闭资源

    • 在完成数据库操作后，必须关闭所有打开的资源，包括 ResultSet、Statement（或 PreparedStatement）和 Connection。这可以通过调用它们的 close() 方法来实现。

三、JDBC 核心接口和类

  1. DriverManager：用于管理数据库驱动程序，建立数据库连接。
  2. Connection：代表与数据库的连接，提供创建 Statement 和 PreparedStatement 的方法，以及事务管理的方法。
  3. Statement 和 PreparedStatement：用于向数据库发送 SQL 语句。
  4. ResultSet：代表数据库查询的结果集，提供访问查询结果的方法。

四、使用 JDBC 的步骤

  1. 导入 JDBC 驱动程序

    • 在项目中添加所需数据库的 JDBC 驱动程序 JAR 文件，并将其导入到项目中。

  2. 加载数据库驱动

    • 使用 Class.forName() 加载数据库驱动程序。

  3. 建立数据库连接

    • 使用 DriverManager.getConnection() 建立与数据库的连接。

  4. 创建 Statement 或 PreparedStatement 对象

    • 根据需要选择创建 Statement 或 PreparedStatement 对象。

  5. 执行 SQL 语句

    • 使用创建的 Statement 或 PreparedStatement 对象执行 SQL 语句。

  6. 处理结果集（如果是查询操作）

    • 如果执行的是查询操作，遍历 ResultSet 对象处理查询结果。

  7. 关闭资源

    • 按照创建的逆序关闭 ResultSet、Statement（或 PreparedStatement）和 Connection。

六、JDBC 事务处理

  1. 开启事务

    • 通过调用 Connection 对象的 setAutoCommit(false) 方法开启事务。

  2. 执行 SQL 语句

    • 在事务中执行一系列的 SQL 语句。

  3. 提交事务

    • 如果所有的 SQL 语句都执行成功，可以调用 Connection 对象的 commit() 方法提交事务。

  4. 回滚事务

    • 如果在执行 SQL 语句过程中出现错误，可以调用 Connection 对象的 rollback() 方法回滚事务，撤销所有已执行的 SQL 语句。

七、JDBC 连接池

  1. 连接池的作用

    • 减少建立数据库连接的时间开销，提高性能。
    • 管理数据库连接的数量，避免资源浪费。

  2. 常见的连接池实现

    • C3P0：一个流行的开源连接池。
    • DBCP（Apache Commons DBCP）：另一个常用的连接池。
    • HikariCP：一个高性能的连接池。

使用连接池的步骤：

  1. 导入连接池的 JAR 文件。
  2. 配置连接池参数，如数据库连接 URL、用户名、密码、最大连接数等。
  3. 获取连接池对象，并从连接池中获取数据库连接。

八、JDBC 的优势和注意事项

  1. 优势

    • 提供了一种统一的方式来连接不同的数据库，使得开发人员可以在不修改大量代码的情况下切换数据库。
    • 支持 SQL 语句的执行和结果集的处理，方便进行数据库操作。
    • 可以与各种 Java 开发框架集成，如 Spring、Hibernate 等。

  2. 注意事项

    • 正确处理数据库连接的异常，确保资源的合理释放，避免资源泄漏。
    • 注意 SQL 注入攻击，尽量使用 PreparedStatement 代替 Statement。
    • 在使用连接池时，要合理配置连接池参数，以满足应用程序的性能需求。
    • 定期检查数据库驱动的版本，确保使用的是最新的稳定版本，以获得更好的性能和安全性。
